## v3.0（2026-04-28）

### 重构概述

基于实际排查经验，全面重构诊断流程和核心原则。

### 完全保留的原功能

1. ✅ 前置检查 3 关确认
2. ✅ 止损机制（3次尝试/60秒超时）
3. ✅ 诚实原则（读不到就承认）
4. ✅ 通用日志分析能力

### 新增功能

1. 🆕 Phase-based 流程（更清晰）
2. 🆕 检查点交互（用户确认）
3. 🆕 Agent 专项诊断（6层检查清单）
4. 🆕 量化评分（Darwin 8维度）
5. 🆕 独立诊断手册（references/）
6. 🆕 Phase 5 代码节点修改专项流程（两阶段：假设验证→方案勾选）
7. 🆕 苏格拉底式排查原则（不直接给答案，引导用户验证）
8. 🆕 兜底逻辑保护清单（6类严禁删除的防御性代码）
9. 🆕 代码节点故障模式库（10种常见错误模式）
10. 🆕 Dify 原生工具优先（单节点测试/版本控制/Last run）

### 核心原则新增 4 条（第5-8条）

- **苏格拉底式排查原则**：不直接给答案，先呈现假设树请用户验证
- **修改前备份原则**：提醒用户发布版本、使用单节点测试
- **兜底逻辑保护原则**：严禁删除 catch/else/endsWith 等防御性代码
- **检查点（Iteration Gate）原则**：每个关键步骤后暂停等用户确认

### Phase 0 新增检查点

- **检查点 1.6**：Dify 原生工具检查（单节点测试/Last run/版本控制等）
- **检查点 1.7**：修改前备份提醒（发布版本/单节点测试/验证方案）

### Phase 5：代码节点修改专项流程

- 两阶段设计：阶段一探索（假设验证）→ 阶段二确认（方案勾选）
- 假设树模板：症状层 / 直接原因层 / 根因层，用户逐层验证
- 诊断性提问库：假设不成立时的深挖问题集
- 方案分级：P0最小修复 / P1保守优化 / P1+替换逻辑 / P2架构调整
- 兜底逻辑保护清单：6类严禁删除的防御性代码
- 代码节点故障模式库：10种常见错误模式（症状/根因/修复）
- 修改后验证流程：单节点测试/对比测试/边界测试

### 异常处理更新

- 新增：用户否定假设 → 回到 Phase 5.1 重新构建假设树
- 新增：用户要求跳过验证 → 明确告知风险后确认

### 检查点 1.5（Node.js 环境检查）更新

- 从简单 nvm 检查扩展为多版本管理器检测（nvm/n/fnm/Volta）
- 增加平台差异感知（Windows/Unix）
- 增加项目级版本声明检测（.nvmrc/package.json）
- **【新增】强制版本要求**：Playwright MCP 依赖 Node.js >= 18，低于此版本将阻断自动化流程（非手动模式）
- **【新增】版本过低诊断**：L1 检测层增加版本解析，生成针对性修复建议（升级步骤 / 版本管理器命令 / 手动模式切换）
